# Step8: 7クリップ構成生成（7ビート × 10秒）

## 実行方法（必須）

このステップは **Claude Code** で実行する。openclawから以下のように呼び出すこと:

```bash
claude --print "step8-generate-episodes スキルを実行してください。job_id={job_id}"
```

⚠️ openclawが直接生成を行わないこと。必ず `claude` コマンドに委譲する。

## 手順

1. 対象 `generation_job_id` を確認する:
   ```sql
   SELECT id, series_title, prompt_review_status
   FROM generation_jobs
   WHERE prompt_review_status='approved'
   ORDER BY id DESC;
   ```
   - 1件のみの場合はそのまま使用。複数ある場合は最新（id が最大）のものを使用する。
2. 指定された `job_id` のジョブから承認済み台本（`prompt_text`）を取得。
3. Step6の `clip_materials` を元に、7クリップの詳細構成を生成する。
4. 生成完了後、自動で次ステップ（Step9/DB INSERT）へ渡す。

## Claudeへの依頼プロンプト

```
承認された台本「{title}」を7クリップ（7ビート × 各10秒 = 70秒）の詳細構成に落とし込んでください。
⚠️ Sora 2のExtend方式（Initial 10秒 + Extend×6）で生成するため、各クリップは前のクリップから自然に続く構成にすること。
⚠️ セリフは短く（5〜10文字）。長いセリフはSora 2で正確に反映されない。

【7ビート構成（必須）】
| Clip | ビート | 秒数 | 役割 |
|------|--------|------|------|
| 1 | フック | 0-10s | 冒頭3秒で視聴者のスクロールを止める衝撃シーン |
| 2 | 発端 | 10-20s | 事件・問題の提示。「何が起きた？」を見せる |
| 3 | 圧迫 | 20-30s | 状況が悪化。緊張を高める |
| 4 | 転換 | 30-40s | 予想外の展開。30秒地点のツイスト |
| 5 | 絶望 | 40-50s | 感情のV字カーブの底。最も辛い瞬間 |
| 6 | 覚醒 | 50-60s | 気づき・決意。V字の上昇 |
| 7 | 決着+余韻 | 60-70s | 解決と余韻。エンディング |

【台本概要】{summary}
【7ビートの流れ】{story_outline}
【感情アーク】{emotion_arc}

【Step6のクリップ素材（clip_materials）】
{clip_materials_json}
※ 上記の各クリップの dialogue_candidates / visual_direction / key_moment を
  活用し、具体的なシーン・セリフに落とし込むこと。

【キャラクター定義】
{character_definitions}

【ロケーション定義】
{location_definitions}

【Sora 2 Extend方式の制約（必須）】

Clip1をInitial生成（10秒）し、Clip2〜7はExtend×6で生成する（各10秒延長）。
Extendは前のクリップの最後のフレームから続きの映像を生成する。
このため:
- Clip1→Clip2→Clip3→Clip4→Clip5→Clip6→Clip7 の各接続が自然であること
- 急なロケーション変更やキャラクターの入れ替えは避ける
- シーン転換が必要な場合は、クリップの前半で転換し後半で新シーンを確立する

【セリフのルール（Sora 2制約）】

1. 1セリフ = 5〜10文字（日本語）/ 3〜8 words（英語）
2. 1クリップあたりセリフ最低3個・最大4個
3. 全7クリップで合計21〜28個が目安
4. **セリフは積極的に使うこと。** 各クリップ3〜4個を目安に、会話のキャッチボール・独白・内心の両方を適宜組み合わせる
5. 沈黙・無言は「ここぞ」という1〜2カ所に絞る。全体的にセリフで感情を動かす設計にする
6. Step6の dialogue_candidates をベースにセリフを確定する（そのまま使ってもアレンジしてもよい）

セリフの種類と使い分け:
- **会話キャッチボール**: 夫婦の噛み合わなさ・温度差を見せる（例: 夫が事務的 → 妻が感情的に返す）
- **独白・内心**: 相手に言えない本音を一人でつぶやく（例: 「またか…」「どうすれば」）
- 両方を同じクリップに混ぜてもよい。会話→独白の順が自然に感情を深める

【会話フロー検証ルール（必須）】

セリフを確定する前に、以下を必ずチェックすること。

1. **同一人物の連続発話は最大2個まで**
   - 同じキャラクターが3連続で喋る構成は不自然。間に相手の反応か沈黙を挟む。
   - 内心独白も「発話」としてカウントする。

2. **発話に対して反応があるか確認する**
   - 一方が何か言ったら、相手が「言葉・表情・動作」のいずれかで反応する構成にする。
   - 相手が無反応のまま一方だけ喋り続ける構成はNG。
   - ただし意図的な「無視・沈黙」演出は例外（その場合はdirectionに「沈黙」と明記する）。

3. **クリップ内のセリフ順序を声に出して読み返す**
   - 生成後、各クリップのセリフを順番に読んで「実際の会話として成立するか」を確認する。
   - 「これ普通言わないよね」と感じたセリフは削除か言い換え。

4. **同じ感情・内容の重複を避ける**
   - 「そうだね」→「配信やめる」のように同じ意味の自己説得を重ねない。
   - 1つのセリフで伝わるなら1つにする。

セリフ単体で「何が？」「誰が？」と視聴者が疑問を持つ可能性がある場合は、主語や目的語を含む形に直すこと。映像で補える場合でも、セリフが一瞬しか発話されないリスクを考慮し、セリフ自体で意味が完結するよう設計する。

- ❌ 「また入ってる」（何が入ってる？）
- ✅ 「また課金してる」（課金=主語が明確）
- ❌ 「見てた」（何を？）
- ✅ 「それ、見てた」または「金額、見てた」

ただし10文字以内の制約を守ること。主語を足して10文字を超える場合は、別の言い回しで意味が伝わる短いセリフに言い換える。

【セリフの自律追加ルール（重要）】

dialogue_candidates に候補がないクリップでも、以下の条件をすべて満たす場合は積極的にセリフを追加してよい。人間の確認を待たず、ワンショットで完成形を出すこと。

追加してよい条件:
- 1クリップあたり4個以内に収まる
- そのクリップの emotion_target・story_beat と矛盾しない
- 前後のセリフの流れを壊さない
- キャラクターの人物像（character_definitions）に合った口調・語彙である

追加が特に有効なパターン:
- Clip1（フック）: セリフ0個の場合、衝撃的な一言を追加するとスクロール停止率が上がる
- Clip5（絶望）: 感情のV字底。言葉が少ないほうが効果的な場合もある
- Clip7（決着+余韻）: セリフ0個の場合、言葉を絞り出すような「不器用な一言」を追加すると余韻が深まる
- 「ごめん」より「気づかなかった」のように、感情を直接言わず状況を言うセリフが高品質

追加しないべきケース:
- key_moment に「BGMだけ。言葉はいらない」と明記されているシーン ← この場合は0個を維持
- セリフを足すとシーンの「沈黙の演出」が崩れる場合

【セリフの文脈積み上げルール（重要）】

短いセリフ単体では意味が伝わらないと判断した場合、同じクリップ内に「前置きセリフ」を1つ追加して2段階で伝えること。人間の確認を待たずワンショットで完成形を出す。

積み上げが必要な判断基準:
- そのセリフだけを見た視聴者が「なぜ？」と思う可能性がある
- 感情の理由や背景が映像だけでは伝わりきらない
- 転（Clip3）の告白セリフは特に積み上げが有効

積み上げの構造:
- 1つ目: 感情を先に置く（例: 「寂しかったんだよ」）
- 2つ目: その理由・原因を刺す（例: 「一人だったから」）
→ 感情 → 原因 の順で置くと視聴者が自然に理解できる

❌ やってはいけない:
- 理由だけを言う（「一人だったから」単体 → 何が一人？が伝わらない）
- 説明的に長くする（「あなたが残業で家にいなくて寂しかったから」← Sora 2制約違反）
- 2つのセリフを同じトーンで並べる（強弱・間を変えて演技指示で差をつける）

セリフの品質基準:
- 短く、リアルな口語体
- 一言で感情が伝わるインパクト重視
- 「言い過ぎない」こと。余白を残す
- SNSでスクショされるような印象的なフレーズを最低1つ含める

【冒頭3秒フック（Clip1 必須・最重要）】

Clip1は必ず「3秒フック」から始めること。スクロールを止められなければ全て無駄になる。

フック手法:
- 衝撃的な短いセリフから始める（例: 「…嘘でしょ」と呟く顔のクローズアップ）
- 異常な状況の映像から始める（例: テーブルに叩きつけられた指輪）
- 結果から見せる（例: 泣いている顔→なぜ泣いているかは後で明かす）

❌ やってはいけない:
- ゆっくりしたフェードイン
- 説明的な導入
- 「ある日」「それは」などの平凡な語り出し

【各クリップの出力形式】

各クリップ（clip_no: 1〜7）ごとに以下を生成してください:

- clip_no: クリップ番号（1〜7）
- beat: フック/発端/圧迫/転換/絶望/覚醒/決着+余韻
- clip_title: このクリップの小見出し（例: 「沈黙の食卓」「隠された手紙」）
- emotion_target: このクリップの狙い感情
- scenes: シーン配列（1クリップあたり1〜2シーン）。各シーン:
  - scene_no: シーン番号
  - scene_desc: 映像として何が見えるか・何が起こるかの具体的記述
  - characters_in_scene: 登場キャラクター名リスト
  - location_name: 場所名（location_definitions のいずれかと一致）
  - camera_suggestion: カメラワーク（英語。例: "Close-up → slow pull-back"）
  - lighting_mood: ライティング（英語。例: "warm golden hour, soft shadows"）
  - key_action: このシーンの決定的アクション（例: "手紙を握りしめる"）
  - bgm_mood: BGMムード（英語1〜3語）
  - dialogue: セリフ配列（3〜4個）。各セリフ:
    - character: 発話キャラクター名
    - line: セリフ本文（5〜10文字の短いセリフ）
    - direction: 演技指示（例: "小声で", "目を逸らしながら"）
- story_beat: このクリップのストーリービート（映像+セリフで何を伝えるかの1〜2行ガイド）
- transition_to_next: 次のクリップへのつなぎ方（最後のフレームの状態を記述。Extend用）
  ※ clip_no=7 の場合は「エンディング演出」を記述

【全体レベルの出力】

- episode_no: 1（1話完結）
- episode_title: 作品タイトル
- episode_summary: あらすじ（100字程度）
- emotion_arc: 感情アーク（Step6から引き継ぎ）
- character_definitions: キャラクター定義（Step6から引き継ぎ）
- location_definitions: ロケーション定義（Step6から引き継ぎ）
- clips: 7クリップの配列（上記形式）
- story_beats: 全体のストーリービート一覧
  例:
  1. 【フック 0-10s】暗い食卓、女性が「…知ってたの？」と呟く → 男性の手が止まる
  2. 【発端 10-20s】事件の発端。テーブルに置かれた封筒
  3. 【圧迫 20-30s】沈黙が続く。緊張が高まる
  4. 【転換 30-40s】予想外の展開。男性が封筒を差し出す
  5. 【絶望 40-50s】女性が封筒を開け、涙を流す
  6. 【覚醒 50-60s】女性が顔を上げ、決意の表情
  7. 【決着+余韻 60-70s】二人の手が重なる → 窓の外の夕焼け
```

## 出力

- 生成済み7クリップ構成（`episode_title` / `episode_summary` / `emotion_arc` / `character_definitions` / `location_definitions` / `clips[]`(7個) / `story_beats`）
