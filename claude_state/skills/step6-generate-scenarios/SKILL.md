# Step6: 1話完結台本生成

## インプット形式

以下のいずれかで情報を受け取る:

1. **job_id指定**: `job_id=1` のように指定 → 下記SQLでDBから取得
2. **DBレコード直接提供**: テーブル形式やフィールド値として会話中に貼り付けられたもの → そのまま使用

DBレコード直接提供の場合はSQL取得をスキップし、提供されたフィールド値をそのまま使う。

## DB取得クエリ（job_id指定の場合のみ）

```sql
SELECT
  j.*,
  st.name AS story_type_name,
  hp.name AS hook_pattern_name,
  hp.template AS hook_template
FROM generation_jobs j
LEFT JOIN story_types st ON st.id = j.story_type_id
LEFT JOIN hook_patterns hp ON hp.id = j.hook_pattern_id
WHERE j.id = {job_id};
```

## 手順

1. インプット（job_id or DBレコード）から以下のフィールドを確認する:
   - `situation_text` / `story_type_name` / `hook_pattern_name` / `hook_template` / `emotion_target`
2. **ストーリーテリング4原則**に基づき、台本案を **2〜3案** 生成する。
3. 各案を提示し、**そのままstep7へ進む**。

## ストーリーテリング4原則

**① ワンテーマ・ワンエモーション**
- 70秒で伝えるのは**1つの感情変化のみ**。複数テーマ禁止。
- 何の感情がどう変化するかを明確にする。

**② フック（冒頭3秒）**
- 衝撃的な短いセリフ or 異常な映像で引き込む。
- ✅ 「…嘘でしょ」と呟く顔のクローズアップ、落ちるコーヒーカップ
- ❌ ゆっくりしたフェードイン、説明的な導入

**③ セリフ駆動 + 映像演出**
- 短いセリフ（5〜10文字）で物語を進行。
- 映像・表情・カメラワークで感情を増幅。
- ⚠️ Sora 2制約: 1セリフ = 5〜10文字（日本語）/ 3〜8 words（英語）
- ⚠️ **1シーンにつき短いセリフを最低3回・最大4回入れること。**

**④ 7ビート構成（7クリップ × 10秒）**
- Clip1 フック    (0-10s):  冒頭3秒で視聴者のスクロールを止める衝撃シーン
- Clip2 発端      (10-20s): 事件・問題の提示。「何が起きた？」を見せる
- Clip3 圧迫      (20-30s): 状況が悪化。緊張を高める
- Clip4 転換      (30-40s): 予想外の展開。30秒地点のツイスト
- Clip5 絶望      (40-50s): 感情のV字カーブの底。最も辛い瞬間
- Clip6 覚醒      (50-60s): 気づき・決意。V字の上昇
- Clip7 決着+余韻 (60-70s): 解決と余韻。エンディング

## 台本生成プロンプト（Claude内部で使用）

```
以下の情報を元に、1話完結（70秒）のAIショートドラマ台本案を2〜3案生成してください。
⚠️ 7ビート構成の面白い1話完結の物語を作ること。短いセリフで物語を進め、映像演出で感情を増幅させる。
⚠️ セリフはSora 2で動画生成するため、1セリフ5〜10文字（日本語）/ 3〜8 words（英語）の短さにすること。
⚠️ 1シーンにつき短いセリフを最低3回・最大4回入れること。

【状況】{situation_text}
【物語タイプ】{story_type_name}
【フックパターン（参考）】{hook_pattern_name}: {hook_template}
  ※ フックパターンはあくまで雰囲気の参考。未選択の場合は無視してよい。
【ターゲット感情】{emotion_target}
【長さ】70秒（10秒×7シーン）

【ストーリーテリング4原則（必須）】

1. ワンテーマ・ワンエモーション:
   - 70秒で伝えるのは1つの感情変化のみ。複数テーマ禁止。

2. フック（冒頭3秒）:
   - 衝撃的な短いセリフ or 異常な映像で引き込む。
   - ❌ ゆっくりしたフェードイン、説明的な導入
   - ✅ 「…嘘でしょ」と呟く顔のクローズアップ、落ちるコーヒーカップ

3. セリフ駆動 + 映像演出:
   - 短いセリフで物語を進行。⚠️ 1セリフ = 5〜10文字。
   - ⚠️ 1シーンにつきセリフを3回入れること。

4. 7ビート構成（7クリップ × 10秒）:
   - Clip1 フック    (0-10s):  冒頭3秒で視聴者のスクロールを止める衝撃シーン
   - Clip2 発端      (10-20s): 事件・問題の提示
   - Clip3 圧迫      (20-30s): 状況が悪化、緊張を高める
   - Clip4 転換      (30-40s): 予想外の展開、30秒ツイスト
   - Clip5 絶望      (40-50s): 感情V字カーブの底
   - Clip6 覚醒      (50-60s): 気づき・決意
   - Clip7 決着+余韻 (60-70s): 解決と余韻

各案に以下を含めてください:

【物語レベル】
- title: 作品タイトル（キャッチーで短い）
- summary: あらすじ（100字程度）
- story_outline: 7ビートの流れ（各クリップ1〜2行）
- emotion_arc: 感情アーク（例: 「不信→動揺→衝撃→許し」）
- character_definitions: キャラクター定義（名前、役割、外見の英語描写）
  - 例: "Japanese woman, late 20s, black shoulder-length hair, white blouse, navy skirt"
- location_definitions: ロケーション定義（場所名、外見の英語描写）
  - 例: "modern Japanese café, warm lighting, wooden tables, large window on left"

【各シーン — clip_materials配列（7つ）】
- clip_no: 1〜7
- time_range: 例) 0-10s
- scene_summary: このシーンで何が起きるか（1〜2文）
- emotion_target: このシーンの狙い感情
- dialogue_lines: セリフ一覧（最低3行・最大4行、各5〜10文字）
  - 例:
    1. 「何これ…」
    2. 「信じらんない」
    3. 「どういうこと」
    4. 「嘘でしょ」
- visual_direction: 映像の方向性
- key_moment: 決定的瞬間（映像+セリフの組み合わせ）

【バズ設計の観点】
- 共感: 「あるある」と思わせる具体的シーン・フレーズ
- 議論: コメントしたくなる道徳的グレーゾーン
- 意外性: 予想を裏切る展開
- SNS切り抜きポイント: スクショして共有したくなるフレーズ（最低1つ）
```
