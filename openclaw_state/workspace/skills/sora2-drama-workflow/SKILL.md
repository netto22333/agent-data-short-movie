# Visual Storytelling Workflow（セリフ駆動 × Sora 2 Extend）

1話完結のAIショートドラマ制作ワークフロー。7ビート構成の面白い物語を、短いセリフと映像演出で70秒にまとめる。

**設計原則**: セリフ駆動 + 映像演出。短いセリフで物語を進め、映像・動き・表情・ライティングで感情を増幅させる。衝突の真っ只中（in media res）から始め、説明ではなくアクションで見せる。

**なぜ70秒か**: 各プラットフォームの収益化条件は1分以上（60秒超）。70秒は余裕を持って条件をクリアしつつ、10秒×7クリップで自然にビートが切れる最適な長さ。

## 完全自動モード

**このワークフローは全ステップ人間の介入なしで自動実行する。**

- 各ステップのSKILL.mdに「人間レビュー必須」「承認を待つ」等の記述があっても、**このワークフローから実行する場合はすべて無視して自動で進める**。
- 台本・クリップ構成・プロンプトはエージェントが生成したものをそのまま採用する。
- ストーリータイプ・フックパターンはDBとイベント内容から自動選定する。
- job_id選択は最新の未完了ジョブを自動で選ぶ。なければ新規作成から始める。
- エラーが発生した場合のみ停止し、リトライまたはログを残して次へ進む。

## 動画生成方式: Sora 2 Extend

- **Clip1（フック）**: Sora 2で10秒のInitial生成
- **Clip2（発端）**: Clip1をExtendで+10秒延長
- **Clip3（圧迫）**: Clip2をExtendで+10秒延長
- **Clip4（転換）**: Clip3をExtendで+10秒延長
- **Clip5（絶望）**: Clip4をExtendで+10秒延長
- **Clip6（覚醒）**: Clip5をExtendで+10秒延長
- **Clip7（決着＋余韻）**: Clip6をExtendで+10秒延長
- **合計70秒**の1話完結動画

⚠️ Extendは前のクリップの最後のフレームから自然に続く映像を生成する。そのためクリップ間の映像的連続性が自動的に保たれる。

## ステップ構成（全自動）

0. **【対象ジョブ選択】**（既存ジョブを続行する場合）
   - 最新の未完了ジョブを自動選択する:
     ```sql
     SELECT id, series_title, prompt_review_status, video_generation_status
     FROM generation_jobs
     WHERE final_status != 'completed'
     ORDER BY id DESC
     LIMIT 1;
     ```
   - 未完了ジョブがなければ新規作成（Step1から）

1. **Step1**: `step1-collect-trends` — トレンドキーワード収集（CDPスクレイプ、そのまま全件使用）
2. **Step2**: `step2-extract-event` — イベント10件抽出 → 過去1週間のDB採用データと被らないものを自動選定
3. **Step3**: `step3-generate-situation` — ストーリータイプ・フックパターン自動選定
4. **Step4**: `step4-select-type` — 状況文自動生成
5. **Step5**: `step5-insert-job` — generation_jobs INSERT（episode_count=1, duration_sec=70）
6. **Step6**: `step6-generate-scenarios` — 7ビート構成の台本自動生成
7. **Step7**: `step7-save-approved` — 台本をDB保存（自動承認）
8. **Step8**: `step8-generate-episodes` — 7クリップ構成自動生成（7ビート = 10s×7）
9. **Step9**: `step9-insert-videos` — generated_videos INSERT
10. **Step10**: `step10-prepare-sora2-prompts` — Sora 2プロンプト自動生成（Initial + Extend×6）
11. **Step11**: `step11-sora2-generate` — Sora 2で動画自動生成（CDP操作）
12. **Step12**: `step12-remove-watermark` — ウォーターマーク自動除去
13. **Step13**: `step13-whisper-transcribe` — Whisper文字起こし + 自動校正
14. **Step14**: `step14-remotion-subtitles` — Remotion字幕合成
15. **Step15**: `step15-post-multiplatform` — マルチプラットフォーム自動投稿 + ブラウザページクリーンアップ

## ストーリーテリング5原則（Step6で適用）

1. **ワンテーマ・ワンエモーション**: 1話（70秒）で伝えるのは1つの感情変化のみ
2. **In Media Res**: 説明から始めず、衝突の真っ只中から始める。冒頭3秒で視聴者を掴む
3. **セリフ駆動 + 映像演出**: 短いセリフで物語を進め、映像・表情・カメラワークで感情を増幅させる
4. **7ビート構成**: フック(0-10s) → 発端(10-20s) → 圧迫(20-30s) → 転換(30-40s) → 絶望(40-50s) → 覚醒(50-60s) → 決着+余韻(60-70s)
5. **感情V字カーブ**: 絶望（40-50s）で谷を作り、覚醒→決着で反転させる。この落差がコメント・シェアを誘発する

## セリフの制約（Sora 2対応）

- **1セリフ = 5〜10文字（日本語）/ 3〜8 words（英語）**
- 1クリップあたりセリフ1〜2個（長すぎるとSora 2で反映されない）
- プロンプトでは「キャラクターがこのセリフを話している」として指示
- セリフは物語の核心を突く短い言葉を選ぶ（例: 「嘘でしょ」「知ってたよ」「行かないで」）

## 7ビート × Extend の構成

| クリップ | 秒数 | ビート | Sora2操作 | 内容 | 狙い |
|---------|------|--------|-----------|------|------|
| Clip1 | 0-10s | フック（Hook） | **Initial生成** | 衝突の真っ只中から始める。視覚的衝撃+表情+テロップ | 冒頭3秒で80%の視聴者を掴む |
| Clip2 | 10-20s | 発端（Incite） | **Extend** | 日常が壊れる瞬間。誰が・何が・何が掛かっているかをアクションで見せる | 状況理解＋感情移入 |
| Clip3 | 20-30s | 圧迫（Pressure） | **Extend** | 障害・対立の登場。衝突が深まる | 緊張の上昇 |
| Clip4 | 30-40s | 転換（Twist） | **Extend** | 重大な暴露・発見。主人公の態度が変わる | 驚き＋感情の転換点（30秒リテンション壁突破） |
| Clip5 | 40-50s | 絶望（Abyss） | **Extend** | どん底。最大の逆境。視聴者の共感が最大化 | 感情の谷→次の反転への期待 |
| Clip6 | 50-60s | 覚醒（Turning） | **Extend** | 気づき・決意・反撃開始。希望が見える | カタルシスへの助走（60秒収益ライン突破） |
| Clip7 | 60-70s | 決着＋余韻（Resolution） | **Extend** | 感情の着地 or クリフハンガー。コメント・シェアを誘発 | 満足感 or 「続きが気になる」 |

## ルール

- **全ステップ完全自動。人間の介入・承認・確認は不要。**
- 各ステップSKILL.mdの「人間レビュー必須」「承認を待つ」等の記述はこのワークフローでは無視する。
- **全ステップは選択した job_id に紐づくデータのみを参照すること**。
- **episode_count は常に 1**（1話完結）。
- 全ステップ結果をDB記録し、監査可能性を維持する。
- エラー発生時はログに記録し、リトライ可能なものは自動リトライする。

## Sora 2自動生成の流れ（Step11）

1. **Clip1**: clip1プロンプトをCDP経由でSora 2に投入 → 10秒で生成
2. **Clip2**: 生成されたClip1動画を選択 → Extend → clip2プロンプト投入 → 10秒延長
3. **Clip3**: Clip2まで延長された動画を選択 → Extend → clip3プロンプト投入 → 10秒延長
4. **Clip4**: Clip3まで延長された動画を選択 → Extend → clip4プロンプト投入 → 10秒延長
5. **Clip5**: Clip4まで延長された動画を選択 → Extend → clip5プロンプト投入 → 10秒延長
6. **Clip6**: Clip5まで延長された動画を選択 → Extend → clip6プロンプト投入 → 10秒延長
7. **Clip7**: Clip6まで延長された動画を選択 → Extend → clip7プロンプト投入 → 10秒延長
8. 合計70秒の動画をダウンロード → ブラウザページクリーンアップ
