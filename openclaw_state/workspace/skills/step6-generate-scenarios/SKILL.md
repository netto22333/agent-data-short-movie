# Step6: 1話完結台本生成（人間レビュー必須）

## 実行方法（必須）

このステップは **Claude Code** で実行する。openclawから以下のように呼び出すこと:

```bash
claude --print "step6-generate-scenarios スキルを実行してください。job_id={job_id}"
```

⚠️ openclawが直接台本生成を行わないこと。必ず `claude` コマンドに委譲する。

## 手順

1. `generation_jobs` の対象レコードを取得する。
2. 下記**ストーリーテリング原則**に基づいた1話完結の台本案を2〜3案生成する。
3. 各案を人間に提示する。
4. **人間レビュー承認を得るまで先へ進まない。**

## ストーリーテリング4原則

**① ワンテーマ・ワンエモーション**
- 1話（60秒）で伝えるのは**1つの感情変化のみ**
- 複数テーマ禁止。1つの感情の変容を描ききる

**② フック（冒頭3秒）**
- 冒頭3秒で視聴者を掴む
- 衝撃的な短いセリフ or 異常な映像で引き込む
- ゆっくりした導入は禁止

**③ セリフ駆動 + 映像演出**
- **短いセリフ**で物語を進行させる
- 映像・動き・表情・ライティングで感情を増幅する
- セリフと映像が相互に補完し合う構成にする

**④ 起承転結（4ビート × 15秒）**
- 起(0-15s): 世界観確立、キャラ登場、フック
- 承(15-30s): 関係性深化、問題の顕在化
- 転(30-45s): 転換点、衝撃、予想を裏切る展開
- 結(45-60s): 解決、感情の着地、余韻

## DB取得クエリ

```sql
SELECT
  j.*,
  st.name AS story_type_name,
  hp.name AS hook_pattern_name,
  hp.template AS hook_template
FROM generation_jobs j
LEFT JOIN story_types st ON st.id = j.story_type_id
LEFT JOIN hook_patterns hp ON hp.id = j.hook_pattern_id
WHERE j.id = {job_id};
```

## Claudeへの依頼プロンプト

```
以下の情報を元に、1話完結（60秒）のAIショートドラマ台本案を2〜3案生成してください。
⚠️ 起承転結のある面白い物語を作ること。短いセリフで物語を進め、映像演出で感情を増幅させる。
⚠️ セリフはSora 2で動画生成するため、1セリフ5〜10文字（日本語）/ 3〜8 words（英語）の短さにすること。

【状況】{situation_text}
【物語タイプ】{story_type_name}
【フックパターン（参考）】{hook_pattern_name}: {hook_template}
  ※ フックパターンはあくまで雰囲気の参考。未選択の場合は無視してよい。
【ターゲット感情】{emotion_target}
【長さ】60秒（15秒×4クリップ = 起承転結）

【ストーリーテリング4原則（必須）】

1. ワンテーマ・ワンエモーション:
   - 60秒で伝えるのは1つの感情変化のみ。複数テーマ禁止。
   - 何の感情がどう変化するかを明確にすること。

2. フック（冒頭3秒）:
   - 冒頭3秒で視聴者のスクロールを止める。
   - 衝撃的な短いセリフ or 異常な映像で引き込む。
   - ❌ ゆっくりしたフェードイン、説明的な導入、「ある日」的な語り出し
   - ✅ 「…嘘でしょ」と呟く顔のクローズアップ、落ちるコーヒーカップ、涙の一滴

3. セリフ駆動 + 映像演出:
   - 短いセリフで物語を進行させる。セリフが感情の核になる。
   - 映像・表情・カメラワーク・ライティングで感情を増幅させる。
   - ⚠️ Sora 2制約: 1セリフ = 5〜10文字。長いセリフは反映されない。

4. 起承転結（4ビート × 15秒）:
   - 起(0-15s): 世界観確立、キャラ登場、フック。一瞬で「何が起きているか」を伝える
   - 承(15-30s): 関係性深化、問題の顕在化。感情を積み上げる
   - 転(30-45s): 転換点、衝撃。視聴者の予想を裏切る展開
   - 結(45-60s): 解決、感情の着地。余韻を残して終わる

各案に以下を含めてください:

【物語レベル】
- title: 作品タイトル（キャッチーで短い）
- summary: あらすじ（100字程度）
- story_outline: 起承転結の流れ（各ビート1〜2行）
- emotion_arc: 感情アーク（例: 「不信→動揺→衝撃→許し」）
- character_definitions: キャラクター定義（名前、役割、外見の英語描写）
  - Sora 2でシーン間の外見一貫性を保つため、外見を固定の英語で記述する
  - 例: "Japanese woman, late 20s, black shoulder-length hair, white blouse, navy skirt"
- location_definitions: ロケーション定義（場所名、外見の英語描写）
  - 例: "modern Japanese café, warm lighting, wooden tables, large window on left"

【起承転結の各ビート — clip_materials配列】
各ビート（4つ）ごとに以下を設計してください。この情報はStep8でクリップ構成を生成する素材として使います。

- clip_no: クリップ番号（1〜4）
- beat: 起/承/転/結
- beat_summary: このビートで何が起きるか（1〜2文）
- emotion_target: このビートの狙い感情（例: 緊張、動揺、衝撃、安堵）
- dialogue_candidates: セリフ候補（2〜3個。各5〜10文字の短いセリフ）
  - ⚠️ 物語の核心を突く短い言葉を選ぶ
  - 例: ["嘘でしょ", "知ってたよ"], ["行かないで", "もう遅い"]
- visual_direction: 映像の方向性（どんな映像・アクション・表情で感情を伝えるか）
- key_moment: このビートの決定的瞬間（映像+セリフの組み合わせで最もインパクトのある瞬間）

【バズ設計の観点】
- 共感: 「あるある」「わかる」と思わせる具体的なシーン・フレーズ
- 議論: 「これってどうなの？」とコメントしたくなる道徳的グレーゾーン
- 意外性: 予想を裏切る展開・視点の転換
- SNS切り抜きポイント: 「スクショして共有したくなるフレーズ」を最低1つ仕込む
```

## 介入点

- 台本案の選択・修正・却下を受け付ける。
- 承認後、次ステップ（step7-save-approved）へ渡す。

## 出力

- 承認済み台本案（`title` / `summary` / `story_outline` / `emotion_arc` / `character_definitions` / `location_definitions` / `clip_materials[]`）
