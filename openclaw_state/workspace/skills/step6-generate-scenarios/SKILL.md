# Step6: 全体シナリオ生成（人間レビュー①必須）

1. `generation_jobs` の最新 pending レコードを取得する。
2. 下記**映像ファースト4原則**に基づいた全体シナリオ案を2〜3案生成する。
3. 各案を人間に提示する。
4. **人間レビュー①承認を得るまで先へ進まない。**

## 映像ファースト4原則

**① ワンテーマ・ワンエモーション**
- 1話（60秒）で伝えるのは**1つの感情変化のみ**
- 複数テーマ禁止。1つの感情の変容を映像で描ききる

**② ビジュアルフック**
- 各話の冒頭3秒は**映像+感情**で視聴者を掴む
- 言語的フック（「実は…」「知ってたのに…」）禁止
- ストライキングな構図・予想外のアクション・劇的なライティングで掴む

**③ 映像主軸 + セリフ補強**
- ストーリーは映像・動き・表情・ライティングが**主軸**
- 映像だけでも大筋が伝わる構成にする
- セリフ/ナレーションは映像を**補強する役割**。Step8でセリフを載せる前提で感情設計する
- 内面の感情はカメラワーク・色彩・空間で表現しつつ、決定的な一言で増幅させる

**④ 感情アーク（簡潔版）**
- Setup(緊張)→Build(高まり)→Peak(頂点)→Land(着地)の**4ビート**で設計
- 各ビートを映像的に表現可能な形で定義する

## DB取得クエリ

```sql
SELECT
  j.*,
  st.name AS story_type_name,
  hp.name AS hook_pattern_name,
  hp.template AS hook_template
FROM generation_jobs j
LEFT JOIN story_types st ON st.id = j.story_type_id
LEFT JOIN hook_patterns hp ON hp.id = j.hook_pattern_id
WHERE j.prompt_review_status = 'pending'
ORDER BY j.created_at DESC
LIMIT 1;
```

## Claudeへの依頼プロンプト

```
以下の情報を元に、AIショートドラマシリーズの全体シナリオ案を2〜3案生成してください。
⚠️ 映像ファーストで設計すること。映像・動き・表情・ライティングが主軸。セリフは映像を補強する役割として設計し、Step8でセリフ/台本を生成する素材を各話ごとに用意する。

【状況】{situation_text}
【物語タイプ】{story_type_name}
【フックパターン（参考）】{hook_pattern_name}: {hook_template}
  ※ フックパターンはあくまで雰囲気の参考。映像で表現可能な形にアレンジすること。未選択の場合は無視してよい。
【ターゲット感情】{emotion_target}
【話数】{episode_count}話（1話完結の場合は1）
【各話の長さ】{duration_sec}秒

【映像ファースト4原則（必須）】

1. ワンテーマ・ワンエモーション:
   - 1話で伝えるのは1つの感情変化のみ。複数テーマ禁止。
   - 何の感情がどう変化するかを明確にすること。

2. ビジュアルフック（各話ごと）:
   - 各話の冒頭3秒は映像+感情で視聴者を掴む。
   - 言語的フック禁止。構図・アクション・ライティングで掴む。
   - 具体的な映像を記述すること（例: 「暗い食卓、家計簿アプリの赤い数字がスマホ画面に光る」）

3. 映像主軸 + セリフ補強:
   - 映像と動きが主軸。映像だけでも大筋が伝わる構成にする。
   - セリフ/ナレーションは映像を補強する役割。Step8でセリフを載せる前提で感情設計する。
   - 感情は表情・動作・カメラワーク・色彩で表現しつつ、決定的な一言で増幅させる。

4. 感情アーク（4ビート）:
   - Setup(緊張)→Build(高まり)→Peak(頂点)→Land(着地) で設計。
   - 各ビートを映像的に表現可能な形で定義すること。

各案に以下を含めてください:

【シリーズレベル】
- series_title: シリーズタイトル
- series_summary: シリーズ概要（100字程度）
- series_outline: 全話の流れ（各話1〜2行。映像で何を見せるかを中心に記述）
- emotion_arc: 4ビート形式の感情アーク（例: 「不安→疑念→衝撃→解放」）
- cliffhanger_design: シリーズの場合のみ。映像で表現可能な引き（言語的でなく映像的な終わり方）
- character_definitions: 主要キャラクター概要
- location_definitions: 主要ロケーション概要

【各話レベル — episodes配列】
各話ごとに以下を設計してください。この情報はStep8でセリフ/台本を生成する際の素材として使います。

- episode_no: 話数番号
- episode_hook: 冒頭3秒の掴み（映像+感情。例: 「暗い食卓、家計簿アプリの赤い数字がスマホ画面に光る」）
- emotion_target: この話の狙い感情（例: 緊張、切なさ、怒り）
- cliffhanger: この話の終わり方（映像+感情）
- script_material: セリフ/台本生成の素材。以下8要素:
  - core_tension: この話の核となるドラマ的緊張（例: 「娘の学費か、自分の夢か」）
  - turning_point: 感情が変わる決定的瞬間（例: 「封筒の中身を見た瞬間、手が震える」）
  - unspoken_subtext: キャラが言葉にしない本音（例: 「本当は怖いのに強がっている」）
  - audience_hook: 視聴者が続きを見たくなる仕掛け（例: 「あの封筒の中身は次話で明かされる」）
  - viral_element: バズ要素。共感/議論/意外性のいずれかを軸に、具体的な仕掛けを記述（例: 「"年収400万の現実"という共感ワードで議論を誘発」）
  - key_phrases: 印象に残るキーフレーズ候補2〜3個。のちにセリフ化する素材（例: ["もう関係ないでしょ", "最初から分かってた"]）
  - dialogue_direction: セリフの方向性（例: 「言い淀みからの本音爆発」「沈黙の後の一言」「本音と建前のギャップ」）
  - emotional_contrast: 感情の対比設計（例: 「怒りで表現される心配」「優しさに見える諦め」「笑顔の裏の絶望」）

【バズ設計の観点】
各話のscript_materialを設計する際、以下を意識すること:
- 共感: 「あるある」「わかる」と思わせる具体的なシーン・フレーズ
- 議論: 「これってどうなの？」とコメントしたくなる道徳的グレーゾーン
- 意外性: 予想を裏切る展開・視点の転換
- 1話に最低1つはバズ要素を仕込むこと
```

## 介入点

- シナリオ案の選択・修正・却下を受け付ける。
- 承認後、次ステップ（step7-save-approved）へ渡す。

## 出力

- 承認済みシナリオ案（`series_title` / `series_summary` / `series_outline` / `prompt_text` / `emotion_arc` / `cliffhanger_design` / `character_definitions` / `location_definitions` / `episodes[]`（各話 `episode_hook` / `emotion_target` / `cliffhanger` / `script_material` 含む））
