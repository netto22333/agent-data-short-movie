入力タイプの仮定: **【設計・実装・事実確認】**

要望は **ワークフローのみ整理**。

DB設計などは含めず、**制作工程だけ**をまとめる。

---

# **AIショートドラマ制作ワークフロー（3〜5話シリーズ）**

  

## **全体構造**

  

制作は **シリーズ単位 → 各話動画 → 投稿** の順で進む。

```
トレンド取得
↓
イベント抽出
↓
状況生成
↓
物語型選択
↓
フック型選択
↓
全体シナリオ生成
↓
人間レビュー①
↓
エピソード分割（3〜5話）
↓
各話プロンプト生成
↓
各話動画生成
↓
人間レビュー②
↓
投稿
↓
実績取得
```

---

# **1. トレンド取得**

  

AIが外部情報からトレンドを取得する。

  

例

```
Google Trends
ニュース
SNSトレンド
```

出力

```
トレンドキーワード
```

例

```
卒業
新生活
花火大会
```

---

# **2. イベント抽出**

  

トレンドから **ドラマ化できるイベント** を抽出する。

  

例

```
卒業
同窓会
花火大会
```

ここで重要なのは

```
ニュース → ドラマ化可能イベント
```

への変換。

---

# **3. 状況生成**

  

イベントを **動画のシチュエーション** にする。

  

例

```
卒業
↓
卒業式の帰り道
教室で最後の時間
誰もいない校舎
```

ここで **物語の舞台** が決まる。

---

# **4. 物語型選択**

  

ストーリーの型を決める。

  

例

```
誤解
秘密
裏切り
告白
後悔
再会
逆転
```

---

# **5. フック型選択**

  

動画の **冒頭フックの型** を決める。

  

例

```
〜を知ってしまった
〜と思ったら
最後の日に〜
〜した結果
実は〜だった
```

---

# **6. 全体シナリオ生成**

  

AIが **シリーズ全体のストーリー** を作る。

  

入力

```
状況
物語型
フック型
感情
話数（3〜5）
```

出力

```
シリーズ全体のストーリー
```

例

```
全4話

1話: 主人公が親友の裏切りを疑う
2話: 誤解が広がる
3話: 真相の手がかり
4話: 真実が分かり結末
```

ここで **結末まで決めることが重要**。

---

# **7. 人間レビュー①（シナリオレビュー）**

  

レビュー対象

```
シリーズ全体シナリオ
各話の構成
結末
```

チェックポイント

```
ストーリーが成立しているか
各話の引きが弱くないか
炎上リスクがないか
```

承認されたら次へ進む。

---

# **8. エピソード分割（3〜5話）**

  

全体シナリオを **各話構造** に分割する。

  

各話で作る内容

```
episode_no
episode_summary
episode_hook
cliffhanger
```

例

```
第1話
hook: 卒業の日に全部奪われたと思った
cliffhanger: 好きな人の手紙

第2話
hook: 親友の裏切りを疑う
cliffhanger: 真相のヒント

第3話
hook: 誤解が揺らぐ
cliffhanger: 本当の理由

第4話
hook: 真実が明らかになる
ending: 後悔と救い
```

---

# **9. 各話プロンプト生成**

  

各話ごとに **動画生成プロンプト** を作る。

  

入力

```
episode_summary
episode_hook
cliffhanger
```

出力

```
動画生成プロンプト
```

---

# **10. 各話動画生成**

  

AI動画生成ツールで **各話動画を生成**。

  

生成される動画数

```
3〜5本
```

（1話ごと）

---

# **11. 人間レビュー②（動画レビュー）**

  

レビュー対象

```
各話動画
```

チェックポイント

```
映像が破綻していないか
フックが成立しているか
シリーズの流れが自然か
投稿して問題ないか
```

問題があれば再生成。

---

# **12. 投稿**

  

各話を順番に投稿する。

  

基本順序

```
episode1
↓
episode2
↓
episode3
↓
episode4
↓
episode5
```

---

# **13. 実績取得**

  

投稿後のパフォーマンスを取得する。

  

例

```
views
likes
comments
shares
```

シリーズ物なので **各話ごとに記録**。

---

# **最終ワークフロー（シンプル版）**

```
1 トレンド取得
2 イベント抽出
3 状況生成
4 物語型選択
5 フック型選択
6 全体シナリオ生成
7 人間レビュー①
8 エピソード分割（3〜5話）
9 各話プロンプト生成
10 各話動画生成
11 人間レビュー②
12 投稿
13 実績取得
```

---

# **このワークフローの本質**

  

制作の基本構造はこれ。

```
AIがストーリー設計
↓
人間がチェック
↓
AIが動画生成
↓
人間がチェック
↓
投稿
```

つまり初期フェーズでは

```
AI制作ライン + 人間承認
```

のハイブリッド制作。

---

もし次に進めるなら、

**このワークフローを実装用のタスク（開発WBS）に分解**すると、実装がかなり楽になる。
